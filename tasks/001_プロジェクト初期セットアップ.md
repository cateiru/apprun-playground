# 001: プロジェクト初期セットアップ

## 概要

Go プロジェクトの基本的なセットアップを行います。Go モジュールの初期化、.gitignore の作成、README.md の更新を含みます。

## 実装内容

### 1. Go モジュールの初期化

```bash
go mod init github.com/cateiru/apprun-playground
```

- モジュール名は適宜プロジェクトの GitHub URL に合わせて変更してください

### 2. .gitignore の作成

以下の内容で `.gitignore` ファイルを作成します：

```gitignore
# Binaries for programs and plugins
*.exe
*.exe~
*.dll
*.so
*.dylib

# Test binary, built with `go test -c`
*.test

# Output of the go coverage tool
*.out

# Dependency directories
vendor/

# Go workspace file
go.work

# Environment variables
.env
.env.local

# IDE
.vscode/
.idea/
*.swp
*.swo
*~

# OS
.DS_Store
Thumbs.db

# Build output
dist/
build/
bin/
```

### 3. README.md の更新

プロジェクトの説明を追加します：

```markdown
# apprun-playground

さくらインターネットの AppRun にデプロイするための検証用リポジトリです。

## 概要

- Go と Echo フレームワークを使用した API サーバー
- `GET /` にアクセスすると `{"status": "ok"}` を返します
- GitHub Actions で自動的に ghcr.io に Docker イメージを push します

## 開発環境

- Go 1.23+
- Docker

## ローカル実行

```bash
go run main.go
```

## Docker ビルド

```bash
docker build -t apprun-playground .
docker run -p 8080:8080 apprun-playground
```
```
